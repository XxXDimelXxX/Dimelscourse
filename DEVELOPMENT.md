# Dimel's School - MVP Development Guide

## Тестовые данные для входа

### Админ
- **Email:** `admin@test.com`
- **Password:** любой (для демо)
- **Доступ:** Админская панель + полный доступ к курсу

### Обычный пользователь
- **Email:** любой другой email (например, `user@test.com`)
- **Password:** любой (для демо)
- **Доступ:** Студенческий интерфейс (курс нужно купить)

## Основные маршруты

### Публичные страницы
- `/` - Лендинг с регистрацией/входом
- `/course/:courseId` - Просмотр информации о курсе (открыт для всех)
- `/purchase` - Страница покупки курса

### Защищенные страницы (требуется авторизация)
- `/dashboard` - Личный кабинет студента
- `/course/:courseId` - Просмотр уроков (с контролем доступа)

### Админская панель (требуется роль admin)
- `/admin` - Главная панель админки
- `/admin/users` - Управление пользователями
- `/admin/payments` - История платежей
- `/admin/course` - Редактирование курса

## Сценарии для тестирования

### Сценарий 1: Новый пользователь без доступа
1. Зайти на `/`
2. Зарегистрироваться с любым email (НЕ admin@test.com)
3. Попадаете в `/dashboard`
4. Видите сообщение "Нет доступа к курсу"
5. Нажмите "Купить курс" → `/purchase`
6. Выберите способ оплаты и оплатите
7. После успешной оплаты получаете доступ
8. Редирект в `/course/fullstack-web-dev`

### Сценарий 2: Админ
1. Войти с email `admin@test.com`
2. В хедере появится кнопка "Админка"
3. Перейти в `/admin`
4. Доступны все страницы управления

### Сценарий 3: Обучение
1. Войти как пользователь с доступом
2. В `/dashboard` нажать "Продолжить"
3. В курсе выбрать урок
4. Нажать "Завершить урок"
5. Прогресс обновится
6. Можно перейти к следующему уроку

## Структура данных (Mock)

### User
```typescript
{
  id: string;
  email: string;
  name: string;
  role: "student" | "admin";
  hasAccess: boolean; // доступ к курсу
  currentLesson: number; // индекс текущего урока
  completedLessons: number[]; // массив завершенных уроков
  progress: number; // процент завершения курса
}
```

### Course
- Один основной курс: "Full Stack Web Development"
- 4 модуля
- 16 уроков всего
- Уроки locked до тех пор, пока не куплен доступ
- Последовательное прохождение

### Payment States
- `pending` - В обработке (ожидание webhook)
- `success` - Успешно (доступ выдан)
- `failed` - Ошибка (доступ не выдан)

## Webhooks (для backend разработчика)

На фронте реализованы UI состояния для webhook событий:

### Success webhook
- Триггер: Платеж успешно обработан
- Действие: `grantAccess()` - выдача доступа пользователю
- UI: Показать success message, редирект в курс

### Pending webhook
- Триггер: Платеж в обработке
- Действие: Показать статус "В обработке"
- UI: Spinner, уведомление о проверке платежа

### Failed webhook
- Триггер: Платеж отклонен
- Действие: Показать ошибку
- UI: Error message, предложить попробовать снова

## Следующие шаги для backend интеграции

1. **Замените моки в AuthContext** на реальные API вызовы:
   - POST `/api/auth/login`
   - POST `/api/auth/register`
   - GET `/api/auth/me`

2. **Интеграция платежей**:
   - POST `/api/payments/create-checkout`
   - Webhook endpoint для обновления статуса
   - GET `/api/payments/status/:id`

3. **API курса**:
   - GET `/api/course` - получить структуру курса
   - POST `/api/progress/complete-lesson` - отметить урок
   - GET `/api/progress/user/:userId` - получить прогресс

4. **Админка**:
   - GET `/api/admin/users` - список пользователей
   - PUT `/api/admin/users/:id/access` - управление доступом
   - GET `/api/admin/payments` - история платежей
   - PUT `/api/admin/course` - обновление курса

## Локальное хранилище

Текущее состояние пользователя хранится в `localStorage` под ключом `user`.
Это временное решение для MVP. В продакшене используйте JWT токены.
