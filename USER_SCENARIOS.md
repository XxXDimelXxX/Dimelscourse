# Сценарии использования Dimel's School MVP

## 🎭 Сценарий 1: Новый студент покупает курс

### Шаги:
1. **Пользователь заходит на главную страницу** (`/`)
   - Видит лендинг с информацией о платформе
   - Видит форму регистрации/входа

2. **Регистрируется**
   - Переключается на вкладку "Регистрация"
   - Вводит имя: "Иван"
   - Вводит email: "ivan@example.com"
   - Вводит пароль
   - Нажимает "Зарегистрироваться"

3. **Попадает в личный кабинет** (`/dashboard`)
   - Видит приветствие: "Добро пожаловать, Иван!"
   - Видит большой баннер "Доступ закрыт"
   - В баннере информация о курсе
   - Кнопка "Купить курс от $59/мес"

4. **Переходит к покупке** (`/purchase`)
   - Выбирает способ оплаты:
     - ☑ Разовая оплата $299
     - ☐ Подписка $59/мес
   - Заполняет данные карты (mock)
   - Нажимает "Оплатить $299"

5. **Обработка платежа**
   - Видит статус "Обработка платежа..."
   - Через 2 секунды: "Оплата успешна!"
   - Сообщение: "Доступ к курсу открыт"

6. **Автоматический редирект в курс** (`/course/fullstack-web-dev`)
   - Видит структуру курса
   - Все уроки первого модуля доступны
   - Может начать первый урок

7. **Начинает обучение**
   - Выбирает первый урок "Введение в HTML"
   - Смотрит урок (mock видео-плеер)
   - Нажимает "Завершить урок"
   - Прогресс обновляется: 1/16 (6%)
   - Автоматически переходит к следующему уроку

8. **Возвращается в личный кабинет** (`/dashboard`)
   - Видит обновленную статистику:
     - Прогресс курса: 6%
     - Уроков завершено: 1
     - Уроков осталось: 15
   - Кнопка "Продолжить" для возврата к обучению

## 🎓 Сценарий 2: Студент продолжает обучение

### Шаги:
1. **Заходит на сайт** (`/`)
   - Вводит email и пароль
   - Нажимает "Войти"

2. **Попадает в дашборд** (`/dashboard`)
   - Видит свой прогресс: 45%
   - Видит "Следующий урок: CSS Flexbox"
   - Нажимает "Продолжить"

3. **Открывается курс** (`/course/fullstack-web-dev`)
   - Автоматически выбран текущий урок
   - Проходит урок
   - Отмечает как завершенный

4. **Прогресс обновляется**
   - Новый процент: 50%
   - Открывается достижение "Половина пути" 🎉

5. **Может посмотреть все завершенные уроки**
   - Уроки с галочкой ✓
   - Следующие уроки доступны для просмотра
   - Будущие уроки заблокированы 🔒

## 👨‍💼 Сценарий 3: Администратор управляет платформой

### Шаги:
1. **Вход как админ**
   - Email: admin@test.com
   - Любой пароль

2. **В хедере появляется кнопка "Админка"**
   - Нажимает на неё
   - Попадает в `/admin`

3. **Просмотр статистики**
   - Видит общую панель:
     - 142 пользователя
     - 89 активных подписок
     - $15,680 общий доход
     - 67% средний процент завершения

4. **Управление пользователями** (`/admin/users`)
   - Видит список всех пользователей
   - Ищет конкретного: вводит email в поиск
   - Видит прогресс каждого студента
   - Может выдать/отозвать доступ кнопкой

5. **Просмотр платежей** (`/admin/payments`)
   - Видит все транзакции
   - Фильтрует по статусу: Успешные/В обработке/Ошибка
   - Видит информацию о webhook событиях

6. **Редактирование курса** (`/admin/course`)
   - Включает "Режим редактирования"
   - Меняет название курса
   - Меняет цены
   - Редактирует названия модулей/уроков
   - Нажимает "Сохранить изменения"

7. **Возвращается к платформе**
   - Нажимает "К платформе" в хедере
   - Попадает обратно в `/dashboard`
   - Может работать как обычный пользователь (у него тоже есть доступ)

## 💳 Сценарий 4: Оплата с ошибкой

### Шаги:
1. **Пользователь пытается купить курс**
   - Заполняет данные карты
   - Нажимает "Оплатить"

2. **Обработка платежа**
   - Статус: "Обработка платежа..."
   - Через 2 секунды: "Ошибка оплаты" (10% вероятность)

3. **Показывается ошибка**
   - Красное уведомление
   - Текст: "Не удалось обработать платеж"
   - "Попробуйте снова или свяжитесь с поддержкой"

4. **Пользователь может:**
   - Попробовать другую карту
   - Выбрать другой способ оплаты (подписка вместо разовой)
   - Связаться с поддержкой

5. **Доступ к курсу не выдан**
   - Возвращается в дашборд
   - Всё ещё видит баннер "Нет доступа"
   - Может попробовать купить снова

## 🔄 Сценарий 5: Webhook обработка (Backend perspective)

### Success webhook
```
POST /webhook/payment-success
{
  "user_id": "user_123",
  "payment_id": "pay_456",
  "amount": 299,
  "type": "one-time"
}
```

**Действия фронта:**
1. Получает событие через API poll или websocket
2. Обновляет `user.hasAccess = true`
3. Показывает success уведомление
4. Редирект в курс

### Pending webhook
```
POST /webhook/payment-pending
{
  "user_id": "user_123",
  "payment_id": "pay_456"
}
```

**Действия фронта:**
1. Показывает статус "В обработке"
2. Опционально: poll статуса каждые 5 сек
3. Ожидает финального success/failed webhook

### Failed webhook
```
POST /webhook/payment-failed
{
  "user_id": "user_123",
  "payment_id": "pay_456",
  "error": "card_declined"
}
```

**Действия фронта:**
1. Показывает ошибку
2. `user.hasAccess` остается `false`
3. Предлагает попробовать снова

## 🎯 Критические пути (Must Work)

### Path 1: Регистрация → Покупка → Обучение
✅ Новый пользователь должен пройти весь путь без багов

### Path 2: Вход → Продолжить обучение
✅ Returning user должен легко продолжить с места остановки

### Path 3: Админ → Управление → Изменения
✅ Админ может управлять платформой без проблем

### Path 4: Оплата → Webhook → Доступ
✅ Процесс оплаты должен корректно выдавать доступ

## 🐛 Edge Cases

### 1. Пользователь открывает урок без доступа
- **Ожидаемое поведение:** Показывается заглушка "Урок заблокирован"
- **Действие:** Кнопка "Купить курс"

### 2. Админ пытается зайти в несуществующий раздел админки
- **Ожидаемое поведение:** 404 страница
- **Действие:** Кнопка "Вернуться на главную"

### 3. Не-админ пытается зайти в /admin
- **Ожидаемое поведение:** Редирект в /dashboard

### 4. Пользователь завершает последний урок
- **Ожидаемое поведение:** 
  - Прогресс = 100%
  - Показывается поздравление
  - Кнопка "Получить сертификат"

### 5. Двойная оплата (пользователь уже купил)
- **Ожидаемое поведение:** 
  - Показывается "У вас уже есть доступ!"
  - Кнопка "Перейти к курсу"
  - Оплата не происходит

## 📱 Основные UI состояния

### Empty States
- ❌ **Нет активности:** "Пока нет активности. Начните проходить уроки!"
- ❌ **Нет доступа:** Большой баннер с призывом купить курс
- ❌ **Нет результатов поиска:** "Пользователи не найдены"

### Loading States
- ⏳ **Обработка платежа:** Spinner + текст
- ⏳ **Загрузка курса:** Skeleton или spinner

### Success States
- ✅ **Оплата успешна:** Зеленое уведомление
- ✅ **Урок завершен:** Галочка + обновление прогресса
- ✅ **Доступ выдан:** Success message

### Error States
- ❌ **Ошибка оплаты:** Красное уведомление
- ❌ **404:** Страница не найдена
- ❌ **Нет доступа:** Заблокированный урок
