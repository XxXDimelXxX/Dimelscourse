# Dimel's School MVP v1.0

Образовательная IT платформа - Минимально жизнеспособный продукт (MVP)

## 🎯 Что реализовано

### ✅ Страницы для пользователей

1. **Лендинг (/)** 
   - Регистрация / Вход
   - Информация о платформе
   - Превью курсов

2. **Личный кабинет (/dashboard)**
   - Статистика прогресса
   - Мои курсы
   - Достижения
   - 2 состояния: с доступом / без доступа к курсу

3. **Страница курса (/course/:id)**
   - Просмотр структуры курса
   - Видео-плеер (заглушка)
   - Модули и уроки
   - Отметка урока как завершенного
   - Автоматический расчет прогресса
   - Locked/Unlocked состояния уроков

4. **Покупка курса (/purchase)**
   - Выбор способа оплаты: разовая / подписка
   - Форма оплаты (mock)
   - 3 состояния: pending / success / failed
   - Автоматическая выдача доступа после успешной оплаты

### ✅ Админская панель (/admin)

1. **Dashboard**
   - Общая статистика платформы
   - Последние платежи
   - Новые пользователи

2. **Пользователи (/admin/users)**
   - Список всех пользователей
   - Фильтрация и поиск
   - Просмотр прогресса обучения
   - Выдача/отзыв доступа к курсу

3. **Платежи (/admin/payments)**
   - История всех транзакций
   - Фильтры по статусу
   - Информация о webhook событиях

4. **Управление курсом (/admin/course)**
   - Редактирование информации о курсе
   - Управление модулями и уроками
   - Изменение структуры (MVP упрощенная версия)

## 🎨 Основные UI состояния

### Для незарегистрированных
- Доступ к лендингу
- Регистрация/вход
- Просмотр информации о курсе (без доступа к урокам)

### Для студентов без доступа
- Личный кабинет с призывом купить курс
- Кнопка "Купить курс" везде
- Информация о курсе без возможности просмотра уроков

### Для студентов с доступом
- Полный доступ к урокам
- Отслеживание прогресса
- Возможность завершать уроки
- Достижения

### Для админов
- Всё что у студентов +
- Доступ к админской панели
- Управление пользователями и платежами
- Редактирование курса

## 📊 Логика прогресса

- Уроки разблокируются последовательно
- Первый урок всегда доступен (при наличии доступа к курсу)
- После завершения урока открывается следующий
- Прогресс рассчитывается автоматически: `(завершенные уроки / всего уроков) * 100%`

## 💳 Процесс оплаты

1. Пользователь выбирает тип оплаты
2. Заполняет данные карты (mock)
3. Нажимает "Оплатить"
4. Статус: `processing` (2 сек)
5. Статус: `success` (90% случаев) или `failed` (10%)
6. При успехе: автоматически выдается доступ
7. Редирект в курс

## 🔄 Webhook симуляция

На фронте подготовлены UI состояния для webhook событий:

- **payment.success** → Выдача доступа к курсу
- **payment.pending** → Показ состояния "В обработке"
- **payment.failed** → Показ ошибки, предложение попробовать снова
